Rezo Hobbylos Habit Tracker

Infos:

Login:
Beim Login überprüft das Backend lediglich die Spotify-Benutzer-ID.
Es werden keine weiteren Daten abgerufen.
Die E-Mail-Adresse benötige ich, um den Account bei Spotify für die Nutzung der API freizuschalten.
Im Hintergrund wird bei jeder Anfrage die Benutzer-ID überprüft.

Speicherung:
Die Daten werden in einer Singlestore-Datenbank gespeichert.
Dies erleichtert es, wenn ich in Zukunft Auswertungen/Statistiken hinzufügen möchte.
Außerdem muss ich für die Datensynchronisierung sowieso eine Datenbank verwenden.

Erreichbarkeit:
Wenn ihr das hier nutzen möchtet, würde ich es von der Subdomain search auf die Hauptdomain verschieben.

Was noch fehlt:
Aktuell würde ich das Design noch etwas verbessern und für die mobile Nutzung optimieren.
Außerdem handelt es sich bisher lediglich um einen Tracker den ihr gemeinsam nutzen könntet.
Wenn ihr jedoch jeder einen eigenen site by site haben möchtet, würde ich das ebenfalls implementieren.
Ansonsten bin ich natürlich offen für jegliche Verbesserungsvorschläge.
